# Мониторинг рекомендательной системы

## Метрики сервиса (Prometheus / `/metrics`)

| Метрика | Тип | Описание |
|---------|-----|----------|
| `recommend_requests_total` | counter | Всего запросов к `/recommend` |
| `recommend_errors_total` | counter | Ошибки при обработке запросов |
| `recommend_cold_start_total` | counter | Запросы, обслуженные popularity-fallback |
| `recommend_latency_seconds` | histogram | Время ответа (корзины: 5–1000 мс) |
| `service_uptime_seconds` | gauge | Время работы процесса |

Эндпоинт: `GET /metrics` — текстовый формат Prometheus.
Быстрый снапшот: `GET /metrics/snapshot` — JSON.

## Качество модели (offline)

Метрики вычисляются при каждом дообучении (Airflow DAG) на holdout-пользователях
(последние 14 дней из окна 90 дней), которые совершили хотя бы одно `addtocart`.

| Метрика | Baseline (popularity) | ALS v1 | Порог для production |
|---------|----------------------|--------|----------------------|
| recall@10 | 0.034 | **0.087** | ≥ 0.010 |
| ndcg@10 | 0.021 | **0.056** | — |
| precision@10 | 0.009 | **0.024** | — |

Улучшение ALS vs popularity: **+156% recall@10**, +167% NDCG@10.

## Дрейф данных

Признаки для отслеживания дрейфа (вычисляются в DAG при загрузке новых событий):

| Сигнал | Как измерять | Порог алерта |
|--------|-------------|--------------|
| Распределение типов событий | доля view/cart/buy | ±20% от базового |
| Количество уникальных пользователей/7d | абсолютное значение | < 50% предыдущего |
| Количество уникальных товаров/7d | абсолютное значение | < 50% предыдущего |
| Средняя длина сессии (events per visitor) | среднее по visitors | ±30% от базового |

Базовое распределение: view 96.7%, addtocart 2.5%, transaction 0.8%.

## Cold-start

Доля cold-start запросов отслеживается через `recommend_cold_start_total`.

- **Ожидаемый уровень**: 60–80% (большинство пользователей новые)
- **Алерт**: > 95% — возможна проблема загрузки модели

## Airflow мониторинг

DAG `ecommerce_recsys_retrain` запускается каждый понедельник.
При recall@10 < 0.010 задача `update_model` завершается с ошибкой — production-модель
остаётся прежней. Алерт приходит через Airflow (email_on_failure или интеграция со Slack).
